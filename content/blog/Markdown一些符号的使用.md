# 基本的语法
## 1.标题、强调
标题，强调分别用`#`和`*`就可以实现
- 其中几个`#`就代表是几级标题
- 一对`*`就是斜体(tilt)，两对`*`就是加粗(bold) 如 *斜体* 和 **加粗**

---
## 2.无序表格
当我们想列出一些关键点时，这被称为无序表格，可以在一行的开始用`-`加一个空格实现，上文中的·其实就是这样
- 123 
	- 112233
	- 445566
	- 778899
- 456
- 789

当`-`后不带空格或不是所在行的开头，不会实现上述效果
上面还用到了嵌套列表，在第一行后回车再按tab即可实现子无序列表

---
## 3.表格
列表功能由`|`和`-`实现
第一行是表头；第二行用|------|------|做分割；第三行起为内容

其效果展示如下：

| 名字  | 成绩  | 排名  |
| --- | --- | --- |
| XX1 | YY1 | ZZ1 |
| XX2 | YY2 | ZZ2 |
| ... | ... | ... |
| XXn | YYn | ZZn |

---
## 4.链接
1.超链接的设置要用到中括号`[]`以及小括号`()`
如下就是一个链接
[wings的博客](https://wings2612.github.io)
代码部分为`[wings的博客](https://wings2612.github.io)`
中括号用来设置名字，是我们自己设置的，但为了不引起歧义以及规范，尽量和小括号中地址指向的内容一致
比如[这不是wings的博客](https://wings2612.github.io)，点进去难道不是我的博客吗?(我求你了orz)

2.另一种方法是直接给出可点击的地址，用`<>`来实现
代码: `<https://bilibili.com>`
点击这个链接<https://bilibili.com>即可导航到b站

3.上面的链接是作为超链接指向网页，我们还可以用来展示图片，不过要额外用一个叹号`!`
假设glby.webp是可被使用的一个图片，若代码为`![图片](glby.webp)`，则会有如下效果即展示这张图片
![哥伦比娅](/images/glby.webp)

## 5.代码展示
上述所有代码展示基本都是用\` 框起来完成的，因为包含一些特殊字符，会被自动渲染，当然希望有一种更为美观和方便的方法
我们要用代码展示代码，而这个代码也很简单，就是是用三个\`将要展示的代码框起来，即\`\`\` .... \`\`\`
进一步，在第一个\`\`\`后加markdown、bash、python等可作为语言标识符，告诉读者这是用哪个计算机语言或者做特殊标记。第一个例子添加了markdown
```markdown
| 名字 | 成绩 | 排名 |
|------|------|------|
| XX1 | YY1 | ZZ1 |
| XX2 | YY2 | ZZ2 |
| ... | ... |...|
| XXn | YYn | ZZn |

```
---

但特别地，对于bash和python这些特殊的，会自动对代码中关键字进行语法高亮，下面是一些例子

添加了python：
```python
for i in range(10):
    print(i)
```

添加了bash：
```bash
git clone https://github.com/user/repo.git
cd repo
npm install
```

## 6.动态图片
动图图片的一个常见格式是gif，但gif的保存格式是像素：放大会模糊；而另一种格式是svg **Scalable Vector Graphics** 矢量图，其更容易编辑，且放大不会模糊，本质是用数学公式来绘图
可以利用AI工具来帮助绘图，或自身有设计能力，可通过在线设计svg的网站上自行绘制。下面就是一个齿轮旋转再停下，并循环往复的动图，格式为svg
![齿轮](/images/齿轮旋转.svg)
值得注意的是，svg格式文件需要被实时读取并执行命令。如pdf是静态页面，不会执行svg的命令；而一般的网页则不会。也就是说，svg依赖于一个能够实时解析和执行SVG动画的渲染环境，而这通常就是浏览器
